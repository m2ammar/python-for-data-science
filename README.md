# Python for Data Science

Self-taught coursework and practice as I learn Python fundamentals and OOP, on my 
way toward Python/ML for data science. This is part of a broader path that started 
with Data Analyst fundamentals (SQL, Excel, Tableau).

## Structure

- **Basic/** — Lectures 1-7 covering core Python fundamentals (variables, control 
  flow, functions, file I/O, etc.), with practice files for each lecture.
- **Practice_Questions/** — 11 standalone practice problems.
- **Python-OOP/** — Lectures 8-9 on Object-Oriented Programming, plus applied 
  practice projects: Bank, Box_Area, School_System, Student_Grade, Vehicle_System.
- **notes/** — Reference notes covering Python fundamentals, OOP concepts, and 
  zipping/slicing.

## What I Learned

Coming into Python with a Java background, most of the friction wasn't logic — it 
was syntax habits I had to unlearn:

- **No parentheses around `if`/`else` conditions.** Java's `if (x > 5)` muscle 
  memory kept showing up; Python just wants `if x > 5:`.
- **Indentation is the block structure**, not `{}`. Spacing isn't a style choice 
  in Python — get it wrong and the code doesn't run.
- **No semicolons.** Small thing, but it took a while to stop typing them out of habit.

Beyond syntax, working through File I/O and then OOP (classes translating differently 
than Java's) was where things actually started clicking — seeing the same OOP 
concepts (encapsulation, classes/objects) expressed with Python's simpler syntax.
